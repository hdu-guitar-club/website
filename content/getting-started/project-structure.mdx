# 项目结构

了解 HDJX Docs 的标准项目结构。

## 目录结构

```
hdjx-docs/
├── app/                    # Next.js App Router
│   ├── docs/              # 文档路由
│   ├── layout.tsx         # 根布局
│   └── page.tsx           # 首页
├── components/            # React 组件
│   └── ui/               # UI 基础组件
├── content/              # MDX 文档内容
│   ├── getting-started/  # 快速开始文档
│   ├── components/       # 组件文档
│   └── guides/           # 指南文档
├── public/               # 静态资源
├── theme.config.tsx      # 主题配置
└── next.config.js        # Next.js 配置
```

## 核心文件说明

### `content/` 目录

存放所有的 MDX 文档文件。每个子目录代表一个文档分类。

### `_meta.ts` 文件

用于定义侧边栏导航结构：

```typescript
export default {
  introduction: {
    title: '介绍',
  },
  installation: {
    title: '安装',
  },
};
```

### `theme.config.tsx`

配置文档站点的主题，包括：

- Logo 和品牌
- 导航链接
- 侧边栏设置
- 目录设置
- 搜索配置
- 深色模式

### `components/` 目录

存放可复用的 React 组件，特别是 UI 组件。

## 路由映射

Nextra 会自动将 `content/` 目录下的 MDX 文件映射到对应的路由：

| 文件路径 | URL |
|---------|-----|
| `content/index.mdx` | `/docs` |
| `content/getting-started/introduction.mdx` | `/docs/getting-started/introduction` |
| `content/components/button.mdx` | `/docs/components/button` |
